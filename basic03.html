HTML構成規則

2017/07/19 現在

基本ルール

特に指定がない限り、以下のフォーマットを使用してください。

UTF-8 BOM無し CR+LF

基本的には、Google HTML/CSS Style Guide に沿って記述します。※一部除く

  • W3Cの規格に準拠した、正しいHTML文書を記述する。
  • コードは小文字で記述する。（alt属性など値が文字列の場合は除く）
  • 指定がない限り、文章中の英数字は半角を使用する。
  • コードのインデント（字下げ）は、タブではなく半角スペース2つ分とする。
  • 必要に応じてコメントを入れる、ブロック要素で改行を入れるなど、ソースを読みやすくしておく。
  • CSSでの再現が困難な場合を除き、レイアウトを目的としたtableコーディングは行わない。
  • 記述ミスを防ぐために、タグの省略はしない。（終了タグなど）
  • CSS、JavaScript、画像、サイト内リンク等のパスはルートパスで記述する。（制作時は相対パスでも可能）
［参考］Google HTML/CSS Style Guideを全部日本語に訳してみた【HTML編】
http://buchineko.website/google_styleguide_html/

DOCTYPE宣言

HTML5での制作を基準とします。
過去に制作した案件はXHTMLでの制作が主流でしたので、更新作業がある場合は注意してください。

HTML5 記述例

過去のものと比べて、すっきりシンプルになっています。
新しいタグが追加され、以前に使われていた一部のタグは使用できなくなりました。


<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="utf-8">
　︙

XHTML 1.0 Transitional 記述例

すべてのタグにおいて、閉じタグ「<… />」が必須となっています。
HTML5から採用されたタグは使用できません。


<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="ja" lang="ja">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<meta http-equiv="Content-Style-Type" content="text/css" />
<meta http-equiv="Content-Script-Type" content="text/javascript" />
　︙

head要素

特別な指定がない限り、以下の順番で要素を記述していきます。
ただし、プログラムで動的に作られるソースコードは、以下の限りではありません。

  1. 文字コード、IE互換モード、viewportの指定
  2. title要素
  3. meta要素（description、keywords）
  4. OGP設定
  5. ファビコン設定
  6. 全ページ共通で読み込む要素
  7. CSS要素
  8. JavaScript要素

1. 文字コード、IE互換モード、viewportの指定

viewportはスマートフォン向けの設定です。状況に応じて設定してください。


<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1">

スマートフォン用の電話番号の自動リンク機能は、無効化しておく方が無難です。


<meta name="format-detection" content="telephone=no">

2. title要素

titleタグは「ページの主題」です。検索結果やブラウザのタイトルバーに表示される、SEO対策で最も重要な要素です。

  • 検索エンジンが最重要視する要素。
  • 検索結果には最大全角34文字程度が表示され、残りは省略される。（GoogleのWeb表示では30文字程度）
  • 上位化した際のクリック率やコンバージョンにも大きな影響を与える。

下階層ページで特に指定がない場合は、「ページタイトル（半角スペース） | （半角スペース）サイトタイトル」で文字列をつないでください。


<title>ページタイトル | サイトタイトル</title>

3. meta要素（description、keywords）

descriptionは検索エンジン向けの「ページの概要説明文」になります。
titleを補足できるような説明を行うことで、クリック率の向上、コンバージョン率に良い影響を与えることができます。

  • 検索結果においてtitleのすぐ下に表示されるので、簡潔で読みやすい文章にする。
  • 検索結果には最大全角120文字程度が表示され、残りは省略される。（スマホは50文字程度）
  • OGP設定していない場合は、descriptionの文章が使われる。
  • SEOキーワードをはじめの方に入れる。（順位向上そのものには大きく影響はしない）
  • すべてのページで違う文章にするべき。（重複するくらいなら記述しない方がよい）

<meta name="description" content="ページ概要">

なお、以前はkeywordsの設定がSEO対策として有効とされていましたが、現在Googleにおいてその効果はなくなりました。
今後、SEOの評価基準の見直しも考えられますが、無理に設定するくらいなら記述しない方が無難です。

  • 多すぎると過剰なアピールと捉えられペナルティの要因となる可能性もあるため、最大でも5個程度に収めたほうが良い。
  • keywordsにて書いたキーワードは本文中でも必ず使用していること。
  • 削除することを推奨しているサイトもあるため、指定がない限りは設定不要とする。

<meta name="keywords" content="キーワード1,キーワード2,キーワード3">

4. OGP設定

OGPとはFacebookやTwitterなどのSNSでシェアされた際に、そのページのタイトル・URL・概要・アイキャッチ画像などを意図した通りに表示させる仕組みです。案件ごとに必要に応じて設定してください。


<meta property="og:title" content="ページタイトル | サイトタイトル">
<meta property="og:type" content="article">
<meta property="og:image" content="http://www.xxxxxx.com/img/xxxxxx.png">
<meta property="og:url" content="http://www.xxxxxx.com/xxxxxx/">
<meta property="og:site_name" content="サイトタイトル">
<meta property="og:description" content="ページ概要">

5. ファビコン設定

ファビコンはICO形式のファイルを「favicon.ico」というファイル名でウェブサーバーのルート直下に配置することでブラウザが自動的に読み込みます。
通常のPCサイトであれば、最低限 16×16、32×32、48×48 のマルチアイコンを制作してルート直下に配置しておきます。

スマートフォン等、マルチデバイス対応を考慮して細かく設定していくと大変なことになりますが、その際は最低限、以下のように設定しておきます。（ルート直下に配置する場合）


<link rel="icon" href="/favicon.ico" type="image/vnd.microsoft.icon">
<link rel="shortcut icon" href="/favicon.ico" type="image/vnd.microsoft.icon">
<link rel="apple-touch-icon" href="/apple-touch-icon.png" sizes="152x152">
［参考］faviconをマルチファビコンとapple-touch-iconの2枚で済ます！
http://www.02320.net/apple-touch-icon_for_android/
［参考］ファビコン作成 favicon.ico 無料で半透過マルチアイコンが作れます。
https://ao-system.net/favicon/
［参考］様々なファビコンを一括生成。
https://ao-system.net/favicongenerator/

6. 全ページ共通で読み込む要素

サイトの全ページに共通で読み込む要素は、まとめて記述してください。（「common.css」CDNに設置されたjQuery「common.js」など）

  • 読み込むjQueryのバージョンは、使用するプラグインの対応状況によって変更する。（最新を使えばよい訳ではない）
  • 全ページ共通の要素であれば「common.css」「common.js」に限定しない。
  • 制作時は相対パス（../css/xxxxxx.css）指定でよいが、公開時はルートパス（/css/xxxxxx.css）で指定する。
  • 外部サイトからのリンクパスはプロトコル「http:」「https:」を省略して記述する。
  • JavaScriptは、特別理由がない限りはhead内に記述する。

<!-- common -->
<link rel="stylesheet" href="/css/common.css">
<script src="//ajax.googleapis.com/ajax/libs/jquery/バージョン/jquery.min.js"></script>
<script src="/js/common.js"></script>

7. CSS要素

デザイン要素であるCSSは、HTML上に直接記述せず、外部CSSから読み込むようにしてください。
やむを得ずHTML上に記述する場合も、タグに直書きはせず、ここに記述してください。


<!-- css -->
<link rel="stylesheet" href="/css/xxxxxx.css">
<style type="text/css">
.hoge {
  margin: 10px;
}
</style>

8. JavaScript要素

JavaScript要素は、可能な限り外部ファイル化してください。
ページのレンダリング速度を考えて、bodyタグの最後に記述した方がよいとされていますが、実際のところ大差はないようです。
JavaScriptを最後にすることで動作が安定せず、逆効果となってしまうこともありますので、基本的にはhead内へ記述してください。
ただし、著しく表示速度が改善される場合や、実行のタイミングを制御したい場合などの都合に応じて、記述場所は変更してもかまいません。


<!-- js -->
<script src="/js/xxxxxx.js"></script>
<script>
$(function() {
  $('.hoge').hide();
});
</script>

head要素まとめ

まとめると、HTMLのベースは以下のようになります。
以下のタグはあくまで一例で、案件によって設定は変わりますので、臨機応変に対応してください。


<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge">
<meta name="viewport" content="width=device-width, initial-scale=1">
<meta name="format-detection" content="telephone=no">

<title>ページタイトル | サイトタイトル</title>
<meta name="description" content="ページ概要">
<meta property="og:title" content="ページタイトル | サイトタイトル">
<meta property="og:type" content="article">
<meta property="og:image" content="http://www.xxxxxx.com/img/xxxxxx.png">
<meta property="og:url" content="http://www.xxxxxx.com/xxxxxx/">
<meta property="og:site_name" content="サイトタイトル">
<meta property="og:description" content="ページ概要">

<!-- common -->
<link rel="stylesheet" href="/css/common.css">
<script src="//ajax.googleapis.com/ajax/libs/jquery/バージョン/jquery.min.js"></script>
<script src="/js/common.js"></script>

<!-- css -->
<link rel="stylesheet" href="/css/xxxxxx.css">

<!-- js -->
<script src="/js/xxxxxx.js"></script>
</head>

<body>
コンテンツ内容
</body>

</html>

補足

状況に応じて設定するべきmeta要素として、以下のようなものがあります。

canonical属性
異なるURLを正規化する際に使用できる。
似た・同じ内容のページを1つに統一することができ、Googleからの評価を1つのページに集約できる。
alternate属性
URLは違うが、内容が同じスマホ・モバイルページがある場合。
検索エンジンの「内容が同じなのにURLが違う＝マイナス評価」ということを防げる。

アウトライン構造

アウトラインは、HTML文書の構造を明示するための仕組みです。
HTML5の仕様では、セクショニング・コンテンツを使用してアウトラインを示すことが推奨されています。
正しい文書構造で記述することで、検索エンジンがページ構造を理解しやすくなります。

  • body要素はアウトラインを形成する。
  • header、footer、main要素はアウトラインを形成しない。
  • レイアウトのためのコンテナ要素として使うものではない。（レイアウトが目的の場合はdivを使う）
  • セクショニング・コンテンツには見出し要素が必要だが、navとasideはなくてもよい。

※main要素はIEで認識されない状況もあるようです。使用する際は注意してください。

セクショニング・コンテンツ

HTML5からは文書構造を正しく理解するために、構造化の為のセクショニングタグが追加されました。
その中でもセクション（文書構造）を定義しているものは次の4つです。

section HTMLドキュメント内の一般的なセクション（章や節）を示す要素。
見出し要素の使用が強く推奨されている。
article サイト内で自己完結しており、それ自体で独立したコンテンツとして成り立つもの。
内容だけ抜き出しても違和感がない、ブログ記事やニュース記事など。
aside コンテンツ内における余談・補足情報のセクションであることを示す要素。
広告、参考リンク、レコメンドなど仮に無くなってもコンテンツに影響がないもの。
nav サイトにとって主要なナビゲーションであることを示す要素。
グローバルメニュー、サイドメニュー、パンくずリストなどが該当する。

https://yukiyuriweb.com/contents-model-in-html5/
［参考］HTML5文書のアウトラインを確認できるサービス「HTML5 Outliner」
https://gsnedders.html5.org/outliner/
※「Untitled Section」と表示されたら文書構造が正しく出来ていません。ただし、navとasideの部分は無視しても大丈夫です。

セクショニング・ルート

セクショニング・ルートは、独自のアウトラインを形成する要素です。
ただし、セクショニング・コンテンツと異なり、文書全体のアウトラインには影響を与えません。
body、blockquote、details、fieldset、figure、td要素が該当します。

暗黙的セクション

HTML5においても見出し要素を使用すると、アウトラインが形成されます。
このアウトラインは「暗黙的アウトライン」と呼ばれ、以下のように定義されています。
ちなみに、「セクショニング・コンテンツ」「セクショニング・ルート」から形成されるものが「明示的アウトライン」です。

  • 見出し要素の記述があれば、アウトラインを生成するセクションの始まりとする。
  • セクション内の最初の見出し要素が、そのセクションの見出しになる。
  • 現セクションの見出しレベルより小さければ、下部のセクションになる。
  • 現セクションの見出しレベルより大きいか同じであれば、新しいセクションを開始する。
  • 見出し要素の番号（レベル）は、明示的セクションを超えて意味を成さない。
  • 暗黙的セクションは、明示的セクションを超えられない。

文書構造さえ正しければ、見出し要素の使用順（h1、h2、h3 …）に制限がなくなります。
しかし、実務では混乱を招きかねないので、理由がない限りは順番通り（h1、h2、h3 …）使用する方が無難です。

HTML5 記述例

HTML5のアウトラインを意識した簡単なテンプレートは以下のようになります。
以下のタグはあくまで一例で、案件によって設定は変わりますので、臨機応変に対応してください。


<body>

<!--/ ===== header START ===== /-->
<header>
  <h1>1-1 見出し</h1>
  <nav>
    グローバルメニュー
  </nav>
</header>
<!--/ ===== header  END  ===== /-->

<!--/ ===== main START ===== /-->
<main>

<article>
  <h2>2-1 見出し</h2>
  ブログ記事など単独で成り立つコンテンツ
</article>

<section>
  <h2>2-2 見出し</h2>
  <section>
    <h3>3-2 見出し</h3>
    コンテンツ
  </section>
</section>

</main>
<!--/ ===== main  END  ===== /-->

<!--/ ===== footer START ===== /-->
<footer>
  <p><small>コピーライト</small></p>
</footer>
<!--/ ===== footer  END  ===== /-->

</body>

XHTML 1.0 Transitional 記述例

補足ですが、XHTML 1.0 Transitional にはセクショニングタグはありません。div要素を使用します。


<body>

<!--/ ===== header START ===== /-->
<div id="header">
  <h1>1-1 見出し</h1>
  グローバルメニュー
</div>
<!--/ ===== header  END  ===== /-->

<!--/ ===== main START ===== /-->
<div id="main">

<div class="article">
  <h2>2-1 見出し</h2>
  コンテンツ
</div>

<div class="section">
  <h2>2-2 見出し</h2>
  <h3>3-2 見出し</h3>
  コンテンツ
</div>

</div>
<!--/ ===== main  END  ===== /-->

<!--/ ===== footer START ===== /-->
<div id="footer">
  <p>コピーライト</p>
</div>
<!--/ ===== footer  END  ===== /-->

</body>
