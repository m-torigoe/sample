コーディング基本方針

2017/07/19 現在

はじめに

このコーディングルールは、今までの経験とインターネット等でリサーチした結果を元に作成しています。
定期的に見直し、情報が古いと判断した場合、改良を加えることがあります。

ファイル（ディレクトリ）の種類と構造

案件によっては指定されている場合がありますので、適宜対応してください。

HTML 文書型はHTML5を基準とする。
※文書型の選択は案件による。過去の案件はXHTMLが主流。
画像ディレクトリ
「/img」		 HTMLファイルから参照される画像ファイル（GIF画像やJPEG画像）を格納する。
格納されるファイルの拡張子は「.gif」「.jpg」「.png」など。
CSSディレクトリ
「/css」		 HTMLファイルから参照されるCSSファイルを格納する。
格納されるファイルの拡張子は「.css」。
JavaScriptディレクトリ
「/js」		 HTMLファイルから参照されるJavaScriptファイルを格納する。
格納されるファイルの拡張子は「.js」。
インクルード
「/inc」		 インクルードファイルを格納する。
格納されるファイルの拡張子は「.html」「.php」など作り方による。
Webフォント
「/fonts」		 Webフォントファイルを格納する。
格納されるファイルの拡張子は「.ttf」「.woff」「.eot」など。
ドキュメント
「/doc」		 各種ドキュメントファイルを格納する。
格納されるファイルの拡張子は「.doc」「.xls」「.pdf」など。
リッチコンテンツ
「/media」		 動画ファイルや音声ファイルなどを格納する。
格納されるファイルの拡張子は「.swf」「.mp3」などの各種メディアファイル。

基本的な記述ルール

特に指定がない限り、以下のフォーマットを使用してください。

UTF-8 BOM無し CR+LF

基本的には、Google HTML/CSS Style Guide に沿って記述します。※一部除く

  • W3Cの規格に準拠した、正しいHTML文書を記述する。
  • コードは小文字で記述する。（alt属性など値が文字列の場合は除く）
  • 指定がない限り、文章中の英数字は半角を使用する。
  • コードのインデント（字下げ）は、タブではなく半角スペース2つ分とする。
  • 必要に応じてコメントを入れる、ブロック要素で改行を入れるなど、ソースを読みやすくしておく。
  • CSSでの再現が困難な場合を除き、レイアウトを目的としたtableコーディングは行わない。
  • 記述ミスを防ぐために、タグの省略はしない。（終了タグなど）
  • CSS、JavaScript、画像、サイト内リンク等のパスはルートパスで記述する。（制作時は相対パスでも可能）

画像について

過度な画像の使用、不必要にファイルサイズの大きい画像の読み込みは、サーバーやブラウザに負担がかかってしまいます。
本文の内容を理解して、img要素での配置が相応しいかを判断し、装飾に関するものはCSSで設定するようにしましょう。
また、img要素で配置した画像には代替テキスト（alt属性）が必要ですが、HTML5ではalt属性に詳細な説明を記述するよう定義されています。

  • CSSで再現できるものは、可能な限りCSSで設定する。
  • 本文には直接関係のない、背景のような装飾を目的とした画像は、CSSで設定する。
  • 本文に関係のある（意味のある）画像は、原則img要素として配置する。
  • 画像には、近辺のテキストに関連した、意味のある代替テキストを記述する。
  • 意味を持たない画像の場合は、空のalt属性（alt=""）とする。

［悪い例］<img src="/img/fuji.png" alt="富士山">
［良い例］<img src="/img/fuji.png" alt="うっすらと雪化粧した富士山の写真。">

主な画像の種類

状況に応じて、最もデータサイズが軽く、きれいに書き出される形式を選択してください。

GIF .gif 単色の画像に向いている ／ データ容量が小さい ／ 透過処理ができる（半透明は不可） ／ 簡単なアニメーションが作れる
JPEG .jpg 写真のような多くの色数を扱う画像に向いている ／ 圧縮率を上げるほどデータ容量は小さく、画質が劣化する
PNG .png W3C推奨 ／ 単色にもフルカラーにも対応できる ／ 透過処理ができる ／ 高品質で圧縮率が高いが、フルカラーはJPEGより重くなりがち ／ Fireworksの元データもpng形式なので注意
SVG .svg ベクターデータなので拡大しても劣化しない ／ 写真、色や形状が複雑なものは不向き ／ Retinaディスプレイと相性がよい ／ 古いブラウザは非対応
［参考］PNG画像を圧縮できるサービス「TinyPNG」
https://tinypng.com/

コメントの記述

ソースを分かりやすくするために、注意書きや記録としてコメントを利用してください。
不要なコメント、長すぎるコメントは控えて簡潔に書きましょう。

共通で存在するような大きなブロック

一目で見分けたいので、始まりと終わりを明確にしてください。


<!--/ ===== header START ===== /-->
<header>
  ヘッダーのタグ
</header>
<!--/ ===== header  END  ===== /-->



<!--/ ===== main START ===== /-->
<main>
  メインのタグ
</main>
<!--/ ===== main  END  ===== /-->



<!--/ ===== footer START ===== /-->
<footer>
  フッターのタグ
</footer>
<!--/ ===== footer  END  ===== /-->

各要素で見分けを付けたいもの

始まり、終わりのどちらかだけでもかまいません。状況に応じて日本語でもかまいません。


<!-- #hoge -->
<div id="hoge">
  タグ
</div>
<!-- /#hoge -->



<!-- .hoge -->
<div class="hoge">
  タグ
</div>
<!-- /.hoge -->

メモとして利用

メモ書きとしても利用できます。意味なく長文を記述することは避けて、簡潔に書きましょう。


<!-- ここにコメントを記述 -->

コメントアウトについて

コメントを利用することで、一時的にタグを非表示にすることができます。
ブラウザ上では表示されませんが、ソースを見れば記述されている内容が分かります。
ユーザーに見せてはいけない内容であれば、コメントアウトではなく削除するようにしてください。


<!--
<p>このタグは隠したい</p>
-->

動的ページ（プログラムによって生成されるページ）でソースとして生成したくない場合は、以下のようにコメントアウトしてください。
動的ページの場合は、通常のコメントアウトではなくこちらを使用してください。


<!--{*
<p>このタグはソースからも隠したい</p>
*}-->
